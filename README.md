# ENS_LUA

проект разработки роботов в "единой среде".
т.к. роботы на LUA используют одни и те же классы и функции, то есть смысл
определить общие и специфичные классы.

пример структуры папок

1. ENS_LUA_Common_Classes
	общие классы для всех роботов. эти классы содержат функции, которые не меняются в роботах, например, создание
	и отправка заявок на сервер, получение информации об инструментах
	
2. ENS_LUA_Strategies
	классы стратегий роботов. 
	эти классы специфичны для каждого вида роботов
	достаточно изменить один файл и изменения применятся ко всем задеплоеным роботам.
	чтобы это произошло, нужно в роботах писать абсолютные пути к классам стратегий, примерно так:

	--common within one strategy
	dofile ("c:\\WORK\\lua\\ENS_LUA_Strategies\\StrategyOLE.lua")	
	
3. ENS_MA_Lua
	репозиторий робота. там ведется разработка одного вида робота.
	
4. 	ENS_MA_Lua\Classes\
			классы, которые  нужны только этому виду роботов.
			как правило, там минимум один класс - settings. в этом классе ведется настройка инструмента, лота и т.д.
	
5. ENS_MA_Lua_RI
	копия папки из п.3. деплой робота для конкретного инструмента. 
	потом не забывать обновлять классы в деплойных копиях из каталога в п.4.
	
	здесь еще возможно разделение продакшн папок на таймфреймы, например
	ENS_MA_Lua_RI_M5 - 5 минут
	ENS_MA_Lua_RI_M60 / ENS_MA_Lua_RI_H1 - 1 час
	ENS_MA_Lua_RI_D  - 1 день
	ENS_MA_Lua_RI_W - 1 неделя
